# 能力评估评分表设计
> 基于 O\*NET Content Model（www.onetonline.org）
> 评分量表：1–7 分制（O\*NET Level Scale）
> 锚点来源：O\*NET Level 2 / 4 / 6（技能/能力）或 Level 1 / 4 / 7（知识）行为描述原文
> LLM 填表规则：从候选人数据中找到最直接行为证据 → 匹配最近锚点 → 在该锚点 ±1 分内输出最终分

> **Mode B（无测试数据）：** 从简历行为证据推断，输出 `score_range: [min, max]`，置信度="低"，status="locked"

## Agent 5 — 工作特质（WorkStylesAgent）

### 评分表概述

| 字段 | 说明 |
|------|------|
| 评估维度 | 4 个子维度，21 个 O\*NET 工作风格项 |

> **Big Five → 工作特质映射规则（有 Big Five 数据时使用）：**
> - N_stable = 7 − (N_original / 100 × 6 + 1)（神经质需反向）
> - 各工作特质分 = Σ(Big Five 维度分 × 权重) + 简历行为修正（±0.5）
> - Big Five 权重详见 agent_config.py WorkStylesAgent 系统提示

## 附录：LLM 统一输出格式规范

### 评分约束规则（全部 Agent 通用）

| 规则 | 说明 |
|------|------|
| 1 | 每个评分项必须输出：score / anchor_level_hit / evidence / confidence |
| 2 | evidence 必须直接引用候选人数据原文片段，禁止编造 |
| 3 | 无证据时固定输出：score=null, confidence="无法判断", evidence="无相关证据" |
| 4 | 分数范围：1.0–7.0（步长 0.5），最多 1 位小数 |
| 5 | anchor_level_hit：填写命中的 O\*NET Level 数字（如 2/4/6 或 1/4/7） |
| 6 | 禁止输出任何表格和评分表以外的自由文字 |
| 7 | 子维度综合分 = 各评分项分数的算术平均（忽略 null 项） |

### 置信度定义

| 置信度 | 定义 |
|--------|------|
| 高 | 有直接、明确的行为证据（如测试数据、具体项目成果） |
| 中 | 有间接证据（如职位描述、经历概括） |
| 低 | 只有很弱的推断信号 |
| 无法判断 | 没有任何相关证据 |
